Lab 09: Git, Doxygen and Beamer

  • Handed out: 26/09

Overview

The days of individually writing useful software programs is over. It's all to be done in a team. git (pronounced with a hard "g") and doxygen (pronounced "docs-ee-gen‚Äù) is therefore important.

Pre-tasks

  1. Precursors to git involve patch and diff. Read the man pages for the diff and patch commands.
  2. The doxygen manual is available at http://www.stack.nl/~dimitri/doxygen/manual/index.html. Use doxygen -g <config-file> to create a simple config file. Also read about Special Comment Blocks in the Doxygen manual to get an idea about how the comments are used.
  3. Go through the following link to learm about basic git commands: http://rogerdudler.github.io/git-guide/ Also spend 15 minutes at try.github.com
  4. Learn to make presentation in Beamer. See for example http://heather.cs.ucdavis.edu/~matloff/beamer.html#tour
  5. Collect all the changes you have made in your Box2D implementation in an area which is owned by the middle roll number in your group. If there are only two members in your group, the middle roll number is that of the lowest roll number.
