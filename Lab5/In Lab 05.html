  • Please write (only if true) the honor code. If you used any source (person or thing) explicitly state it. You can find the honor code on the web page.

The Tasks

Each of the tasks below is best done by reading the reference manual and trying this out yourself. You might find a ready made solution on the net, but it will not be a useful learning experience to copy paste and get the answer. Remember that you are expected to know the item for which you are asking for credit, i.e., marks.
For questions involving arguments, you might find this snippet useful: 
#!/bin/bash
if [[ "$#" != 1 ]]; then
    echo "Usage: $0 argument"
    exit 1
fi
For all cases below, do not look for awkward corner cases but simply consider the most logical input. After figuring out the basic solution you can then try supporting checks for errors.
  1. Learn what happens when you type dmesg at the terminal. Figure out what is it that the dmesg command shows when you plug a (pen) usb drive to your computer. Now try lsusb with the usb drive plugged in. Can you access the contents of the pen drive from the command line? Can you find out how much RAM your computer has using dmesg? If so, report the amount of RAM in your readme.txt file.
  2. Write a script calc.sh to take the number of seconds as input argument and display the equivalent in hh::mm::ss format, where hh=number of hours, mm=number of mins and ss=number of seconds. It should print the message "Not Valid Input" if the argument is not a positive integer.
    Example: 
    calc.sh 3601
1::00::01
  3. You are going to write a primitve editor in this script called myvim.sh. It will take a filename as an argument and append whatever is typed at the standard input to the given file. The operations from the user will end when the user types :q
    Example: 
    % ./my_vim.sh abc
This is a line
THis is a second line
I am now going to type :q
:q
% cat abc
This is a line
THis is a second line
I am now going to type :q
% echo doing it again
doing it again

% ./my_vim.sh abc
This is a third line
:q
% cat abc
This is a line
THis is a second line
I am now going to type :q
This is a third line
  4. Write a script count.sh to count number of words, characters, white spaces (only spaces, no tab or newline) and the special symbols $,<,> when a line of text is entered at the standard input.
    Example: 
    % ./count.sh 
good morn$>ing
Words=2
Characters=14
Spaces=1
Special symbols=2
% ./count.sh 
the quick brown fox jumped over the poor lazy dogs
Words=10
Characters=50
Spaces=9
Special symbols=0
    The output should give the answer in the order specified, and not a random order of Words, Spaces, Characters, etc.
